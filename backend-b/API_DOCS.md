# 백엔드 B API 문서

베이스 URL: `http://127.0.0.1:8000`

---

## 1. 서버 상태 확인

**GET** `/api/b/health`

### 응답 예시
```json
{
  "status": "ok",
  "service": "backend-b"
}
```

---

## 2. 부당계약서 의심알림

**POST** `/api/b/analyze/contract`

### 요청 (Request Body)
```json
{
  "계약서_텍스트": "본 계약은 환불이 불가합니다. 위약금은 잔여금액의 50%입니다.",
  "업종": "헬스장"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| 계약서_텍스트 | string | 계약서 내용 (OCR 또는 직접 입력) |
| 업종 | string | 헬스장 / 필라테스 / 학원 등 |

### 응답 (Response Body)
```json
{
  "위험도": "높음",
  "부당조항_수": 2,
  "탐지된_조항": [
    {
      "원문": "환불 불가",
      "판정": "소비자분쟁해결기준을 위반하여 소비자 권리를 침해함",
      "근거법령": "소비자분쟁해결기준"
    }
  ],
  "총평": "헬스장 계약서에 부당한 조항이 포함되어 있습니다."
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| 위험도 | string | 높음 / 보통 / 낮음 |
| 부당조항_수 | number | 탐지된 부당 조항 수 |
| 탐지된_조항 | array | 부당 조항 목록 |
| 총평 | string | 전체 계약서 요약 |

## 3. 환불요청문구 생성

**POST** `/api/b/generate/message`

### 요청 (Request Body)
```json
{
  "업종": "헬스장",
  "업체명": "강남헬스클럽",
  "결제금액": 300000,
  "이용일수": 12,
  "환불예상금": 240000,
  "부당위약금": 60000
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| 업종 | string | 헬스장 / 필라테스 / 학원 등 |
| 업체명 | string | 업체 이름 |
| 결제금액 | number | 총 결제 금액 (원) |
| 이용일수 | number | 실제 이용한 일수 |
| 환불예상금 | number | 백엔드 A에서 계산된 환불 예상금 |
| 부당위약금 | number | 백엔드 A에서 계산된 부당 위약금 |

### 응답 (Response Body)
```json
{
  "카카오톡용": "강남헬스클럽에서 300,000원의 헬스장 이용권을 구입하여 12일 동안 이용하였습니다. 예상 환불 금액은 240,000원입니다.",
  "이메일용": "제목: 헬스장 이용권 환불 요청\n\n강남헬스클럽 관계자께,...",
  "강경대응용": "제목: 헬스장 이용권 환불 요청 및 부당 위약금 청구\n\n..."
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| 카카오톡용 | string | 간단한 문자/카카오톡 메시지 |
| 이메일용 | string | 정식 공문 형식 |
| 강경대응용 | string | 법조문 인용 + 소비자원 신고 예고 |

---

## 백엔드 A → B 연동 흐름

```
1. 사용자가 계약서 + 결제 정보 입력
2. 백엔드 A: 환불예상금, 부당위약금 계산
3. 백엔드 B: A의 결과값을 받아 문구 생성
4. 프론트: 계산 결과 + 문구 화면 출력
```
